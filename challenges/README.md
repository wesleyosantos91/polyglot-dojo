## 🗺️ Fases de Aprendizado

### Fase 1 — Fundamentos (Desafio 01)
> Construir a base sólida: CRUD, REST, JPA/GORM, Docker, Testes

### Fase 2 — Mensageria & Eventos (Desafios 03–05)
> Comunicação assíncrona: Kafka (producer+consumer), SQS, RabbitMQ

### Fase 3 — Background & Integração (Desafios 06–08)
> Jobs, batch, schedulers, HTTP clients, circuit breakers

### Fase 4 — Protocolos Alternativos (Desafios 09–11)
> Além de REST: gRPC, GraphQL, WebSocket

### Fase 5 — Cross-Cutting Concerns (Desafios 02, 12–13, 15)
> Lambda, Upload S3, Auth/SSO (Keycloak), Cache Redis

### Fase 6 — Produção & Patterns (Desafios 14, 16–18)
> Event Sourcing/CQRS, Elasticsearch, Notifications, API Gateway/BFF

### Fase 7 — Observabilidade & Resiliência (Desafios 19, 26)
> Full OTel stack (Prometheus, Grafana, Tempo, Loki, Jaeger, Pyroscope), resilience patterns

### Fase 8 — AWS & NoSQL (Desafios 20–21)
> DynamoDB Single-Table Design, SNS, EventBridge, Step Functions, Secrets Manager

### Fase 9 — Testing Avançado (Desafios 22–23, 29)
> Testcontainers, BDD/Cucumber, Contract Testing (Pact)

### Fase 10 — Performance, Deep-Dive & Arquitetura (Desafios 24–25, 27–28)
> Virtual Threads/Goroutines, SQL vs ORM, Hexagonal Architecture, Reactive/SSE

## 📏 Regras do Workshop

1. **Todos os desafios** devem ser implementados nas **4 stacks**
2. Docker Compose para dependências de infra
3. Testes automatizados obrigatórios
4. Documentação (README) por projeto
5. Clean code, SOLID, error handling
6. Seguir idiomas de cada stack (não portar 1:1)

## 🚀 Como Começar

```bash
# Escolha um desafio
# Implemente em uma stack por vez
# Valide com os critérios de aceite do desafio
# Marque como concluído na tabela acima
# Repita para as próximas stacks
```

> **Dica:** Comece pelo desafio 01 (CRUD) para estabelecer a base dos 4 projetos, depois siga a ordem das fases ou escolha por interesse.
